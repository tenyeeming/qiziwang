階層式 IP 位址與遮罩Hierarchical IP Addresses & Masks

🏠 Three Parts of an IPv4 Address　IPv4 的三段結構

  • 一個 IPv4 位址（共 32 bits）通常分三段：Network 網路部分 + Subnet 子網部分 + Host 主機部分。
  • Network 部分由號碼登記機構（ISP/IANA）配給公司；公司再自行把剩下的 bits 切成 Subnet 與 Host。
  • ⚠️ 問題：光看位址看不出三段各占幾 bits，只知道總和是 32 bits → 解法是「遮罩 mask」。
三段相加 = 32 bits，但各段長度需靠遮罩才能得知

🎭 What Is a Mask?　遮罩是什麼

  • 遮罩 = 一連串開頭的 1、後面接一連串的 0，總共 32 bits。
  • 像噴漆模板（stencil）：mask bit = 1 的位置「讓 IP 的 bit 通過」；mask bit = 0 的位置「一律變 0」。
  • 表示法：點分十進位（如 255.255.0.0）或前綴（如 /16，代表開頭有 16 個 1）。
遮罩二進位點分十進位前綴
網路遮罩11111111 11111111 00000000 00000000255.255.0.0/16
子網遮罩11111111 11111111 11111111 00000000255.255.255.0/24
🧩 八個 1 = 255
每組 8 個 1 在十進位就是 255；8 個 0 就是 0。所以 /16＝兩組 255 接兩組 0＝255.255.0.0。

遮罩運算：bitwise-ANDApplying a Mask — 算出網路位址

➗ The AND Rule　AND 規則（必背）🔥 必考計算

IP bitMask bit結果

📐 一句話規則

Mask bit = 1 → 保留 IP 原值；Mask bit = 0 → 結果一律為 0。

💡 8-bit 邊界的捷徑
遮罩那一段是 255（全 1）→ 該 octet 原封不動；遮罩是 0（全 0）→ 該 octet 變 0。只有「不是 255 也不是 0」的那一段才需要動手算二進位。

🧮 Worked Example (textbook)　課本範例：跨 8-bit 邊界

3.143.12.12 AND 255.248.0.0 = ?（第 2 段 248 不是 255 也不是 0，要算）

143 = 10001111
248 = 11111000 ← 遮罩
AND = 10001000 = 136
結果: 3.136.0.0（第1段3保留、第3/4段遮罩為0→0）

路由器運作：三步驟Router Operation — A Routing Decision

🔀 Switch vs Router　交換器 vs 路由器（觀念對比）考過 (填充)

項目乙太交換器 Switch路由器 Router
拓撲階層式（hierarchy）網狀（mesh，多條替代路徑）
任兩主機間路徑數只有一條（one）可能多條（可靠性）
表的每個位址交換表每位址只有 一列路由表可能 多列符合，要挑最佳
表的單位單一 EUI-48(MAC) 位址一段 IP 位址範圍（destination + mask）
成本低（switching 便宜）高（routing 較貴）
🎯 考點
「In a hierarchical Ethernet LAN, the number of paths between any two end hosts is ____」→ one（所以交換表每個位址只有一列）。

🪜 Three Steps of a Routing Decision　路由決策三步驟

  1. Step 1 找出所有符合的列（All Row Matches）：對每一列，把該列遮罩套到封包目的 IP（bitwise-AND），結果若等於該列的 Destination 欄 → 該列符合。必須檢查全部的列（可能多列符合）。
  2. Step 2 找最佳符合列（Best-Match Row）：基本規則＝最長前綴（遮罩開頭 1 最多）；若最長前綴相同 → 用 metric 決勝。
  3. Step 3 依最佳列送出：從該列的 Interface（埠）送出，送往 Next-Hop Router；若 Next-Hop 欄是 Local，代表目的主機就在該介面網段上，直接送達。
💭 小題（課本 p40）
若有 127,976 列、只有第 2 與第 7 列符合：① 路由器先檢查哪一列？→ 第 1 列（從頭逐列）。② 要檢查幾列？→ 全部 127,976 列（不能提早停）。

最長前綴 + Metric 決勝Longest-Match & Tie-Breaking — 全章最常考

🏆 The Rules　判斷規則🔥 三卷都考

  • 基本規則：在所有符合的列中，選 遮罩最長（前綴 /N 最大） 的那一列。例：/24 勝 /16。
  • 決勝（Tie-Breaker）：若最長前綴有多列相同 → 比 metric：
    　・metric 是 speed（速度） → 取 數值大 的（越快越好）。
    　・metric 是 cost（成本） → 取 數值小 的（越便宜越好）。
⚠️ 最易錯的地方
先比「前綴長度」，前綴一樣長才比 metric；而 metric 是速度就取大、是成本就取小，方向別搞反。

ARP 位址解析協定Address Resolution Protocol

🔍 Why ARP?　為什麼需要 ARP考過 (填充)

  • 路由器要把封包送給「下一跳路由器」或「目的主機」，從路由表知道對方的 IP 位址。
  • 但封包要裝進該網段的 frame 才能送出，而路由器 不知道對方的資料連結層位址（MAC / EUI-48）。
  • 解法：用 ARP（Address Resolution Protocol），已知 IP → 查出 MAC。
① 廣播ARP Request
「IP 10.19.8.17
你的 EUI-48 是?」
→
② 單播ARP Response
「我的 EUI-48 是
A7-23-DA-95-7C-99」
→
③ 存快取ARP Cache
IP↔MAC 配對
下次免再問
📝 重點
ARP Request 是廣播（問整個網段）；ARP Response 是單播回覆。問到的 IP↔MAC 配對存進 ARP Cache，同一位址不必再跑一次 ARP。
📄 【卷三 A10】If a router knows a target's IP address, it may use ____ to find the target's MAC address.考古
▶ 看答案
ARP（Address Resolution Protocol）。已知 IP 求 MAC。

⏱️ TTL — Time to Live　存活時間🔥 考過3次

  • 發送端設定初值（通常 64 ~ 128）。
  • 路徑上 每經一台 router 減 1；減到 0 的路由器把封包丟棄，並可能回送一個 ICMP 錯誤訊息。
  • 目的：防止封包在網路中無限繞圈（routing loop）。
每跳減 1，歸零即丟，避免封包無限繞行

表示法點分十進位 192.0.2.116 進位、冒號分隔 8 段

✂️ Shortening Rules　縮寫規則（4 條）🔥 必考展開

  1. 把 128 bits 每 4 bits 轉成 1 個 16 進位符號（小寫），4 個符號為一段，共 8 段，段間用冒號 :。
  2. 去掉每段「開頭」的 0（leading zeros）。例：0027→27、0fca→fca、0000→0。
  3. 若有「一整段連續全為 0」的區塊，可用 :: 取代（只留外側冒號）。
  4. 若有多個連續全 0 區塊，只能對「最長」那一個用 ::；若長度平手，對第一個用。其餘區塊不可省。
🎯 考試是反過來：把縮寫「展開」成完整 8 段
看到 :: → 它代表「把缺的段補成全 0、湊滿 8 段」。先數出現有幾段，8 減掉就是 :: 代表幾段全 0。每段再補滿 4 位（前面補 0）。

IPv6 標頭與延伸標頭IPv6 Header & Extension Headers

📋 IPv6 Header (always 40 octets)　IPv6 標頭固定 40 octets

欄位說明
Version值 = 6（0110）
Traffic Class / Diff-Serv服務品質 QoS
Flow Label（20 bits）標記封包屬於某一特定「流（flow）」
Payload Length標頭後面內容的長度（octet）
Hop Limit和 IPv4 的 TTL 功能完全相同（每跳減 1，歸零丟棄）

🆚 IPv6 拿掉了什麼
① 沒有 Fragmentation（封包太大就丟，由發送端先測 MTU）；② 沒有 Header Checksum（交給傳輸層管錯誤，減少每台 router 的處理時間）。

🔗 Hop Limit & Next Header　兩個必考欄位🔥 考過多次

Hop Limit（= IPv4 TTL）

課本原話：「IPv6 Hop Limit works exactly like the Time-to-Live field in IPv4.」名字換了但功能一樣。

  • IPv6 有許多 延伸標頭（extension headers），以 Next Header 欄位一個串一個（daisy chain）。
  • 路徑上的 router 通常只需看 Hop-by-Hop Options Header → 減少每封包處理時間。

TCP 與 UDPTransport Layer — TCP & UDP

🧩 TCP Process & Sequence Number　TCP 運作與序號

  • TCP 收到應用訊息 → 切成 segments（區段），每段裝進一個 IP 封包送出。
  • 每個 segment 放一個 Sequence Number（序號）；接收端用序號把訊息重組、偵測遺失。
  • 接收端正確收到後回送 acknowledgement（ACK，確認），內含 acknowledgement number 指出在確認哪一段。
  • TCP 是 連線導向（connection-oriented）：有明確的開啟與關閉。

🤝 Three-Way Open & Four-Way Close　三次握手 / 四次揮手🔥 考過2次

開啟：SYN → SYN+ACK → ACK（3 次）；關閉：FIN → ACK → FIN → ACK（4 次）
  • SYN 同步的是什麼？ → 雙方的 初始序號（initial sequence number, ISN）。
  • 為何關閉要四步？ → TCP 是全雙工，兩個傳輸方向獨立；一方送完 FIN 時另一方可能還在傳，所以每個方向各用一組 FIN/ACK 分開關閉 → 共 4 步。
📄 【2020 B4】(1) SYN means synchronize—what is synchronized? (2) four-way closing? (3) Why four steps?2020
▶ 看完整解答
(1) 同步雙方的「初始序號（ISN）」。
(2) 四次揮手：FIN→ACK→FIN→ACK，兩方向各自關閉。
(3) TCP 全雙工、兩方向獨立；一方送完(FIN)時另一方可能還在傳，故每方向各一組 FIN/ACK，共四步。
📄 【卷三 B8】(a) Three-way opening of a TCP connection (b) Four-way closing.考古
▶ 看答案
(a) SYN → SYN+ACK → ACK（同步雙方初始序號後建立連線）。
(b) FIN → ACK → FIN → ACK（全雙工，兩方向各自關閉）。
📄 【109-1 B5(1)】解釋 TCP 的 Sequence Number。109-1
▶ 看答案
Sequence Number：每個 TCP segment 的序號，接收端用來重組成原訊息、偵測遺失。

📦 UDP & Ports　UDP 與連接埠

  • UDP 不切片、不需序號/確認 → 簡單輕量；但整個應用訊息必須塞進單一 UDP datagram（最大 65,536 octets）。
  • UDP 訊息叫 datagram；TCP 訊息叫 segment。
  • 多工伺服器用 port number 區分不同應用；ephemeral（暫時）埠 IANA 建議 49152–65535。
🧩 TCP vs UDP 一句話
TCP＝掛號信（可靠、要確認、會重送、有開關連線）；UDP＝明信片（快、輕、不保證到，適合即時語音/影音）。

互動測驗Self-Check Quiz — 點選作答，立即看詳解

🧪 隨堂測驗（10 題）

1. 10.34.65.117 AND 255.255.240.0 = ?
✅ 65=01000001、240=11110000，AND=01000000=64；第4段&0=0 → 10.34.64.0。
2. IPv6 的 ____ 欄位每經一台路由器減 1，功能等同 IPv4 的 TTL。
✅ Hop Limit，課本明說「works exactly like TTL」。
3. 把 "fe80::c2f:a4:300" 展開成完整 8 段，:: 代表幾段全零？
✅ 現有 fe80、c2f、a4、300 共 4 段，8−4=4 段全零。展開 fe80:0000:0000:0000:0000:0c2f:00a4:0300。
4. 路由表中有多列符合，最長前綴又平手（都 /18），metric 是「速度」，該選哪列？
✅ metric 是速度 → 取數值最大（最快）。若是 cost 則取最小。
5. 在階層式乙太網路 LAN 中，任兩台主機之間的路徑數是？
✅ one。因為是嚴格階層、無迴圈，所以交換表每個位址只有一列。
6. 路由器已知下一跳的 IP，但要送 frame 需知道對方 MAC，會用哪個協定？
✅ ARP：已知 IP 求 MAC。廣播 Request → 單播 Response → 存 ARP Cache。
7. TCP 的 SYN「同步」的是什麼？
✅ 同步雙方的初始序號 (initial sequence number)，之後用序號重組與偵測遺失。
8. TCP 連線關閉為何需要「四次」？
✅ TCP 全雙工，兩方向獨立；一方送完 FIN 時另一方可能還在傳，故每方向各一組 FIN/ACK，共四步。
9. IPv4 標頭的 Protocol 欄位，值 6 代表？
✅ 1=ICMP、6=TCP、17=UDP。（IPv6 的 Next Header 也用同一套；ICMPv6=58）
10. 下列關於 IPv6 何者正確？
✅ IPv6 位址 128 bits、16 進位表示、標頭固定 40 octets，且拿掉了分片與標頭檢查碼以加速路由。

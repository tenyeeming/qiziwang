🆕 SMTP Command Order　SMTP 指令順序（2025 B6）🆕 新卷

HELO → 打招呼 MAIL FROM: → 指定寄件者 RCPT TO: → 指定收件者 DATA → 宣告開始送內文 （送出內文） . (dot) → 單獨一個句點 <CRLF>.<CRLF> 結束訊息 QUIT → 結束連線
📄 【2025 B6】用 .(dot)、DATA、HELO、MAIL FROM、QUIT、RCPT TO 解釋寄信流程。2025
▶ 看答案
HELO → MAIL FROM: → RCPT TO: → DATA →（內文）→ .(dot 結束訊息) → QUIT。

🆕 SMTP / POP / IMAP / RFC 822　四者解釋（2023 B6）🆕 新卷

名稱角色說明
SMTP傳遞（寄）在郵件伺服器間「寄送」郵件，近乎即時
POP / IMAP傳遞（收）收件者下載/收取郵件；IMAP 同步電腦與主機、POP 下載到本機
RFC 822/2822文件格式規定郵件「格式」的標準（純文字訊息）
🔑 傳遞 vs 文件
SMTP、POP、IMAP 是「傳遞標準」；RFC 822/2822、HTML 是「文件格式標準」。送得到 + 看得懂，缺一不可。
📄 【2023 B6】Explain SMTP, POP/IMAP, and RFC 822/2822.2023
▶ 看答案
SMTP：伺服器間寄送郵件。POP/IMAP：收件者收信——IMAP 同步電腦與主機、POP 下載到本機。RFC 822/2822：規定郵件格式的文件標準。

應用架構三型Application Architectures

🏗️ Three Architectures　三種架構

  • Stand-alone（獨立）：單機運作。
  • Client/Server（主從）：伺服器提供服務給用戶端。
  • Peer-to-Peer (P2P)：用戶端互相提供服務，伺服器不用或只做次要工作（如幫忙找對方）。
架構桌面處理能力需求
Terminal-Host（終端-主機）桌面無處理能力
Client/Server桌面有一些處理能力
P2P桌面有大量處理能力（靠桌機算力成長才可能）

SQL Injection & XSSWeb 應用兩大攻擊

💉 SQL Injection　SQL 注入考過

  • 在輸入欄填入 SQL 片段，使查詢語意被竄改。
  • 經典手法：帳號/密碼輸入 ' OR '1'='1，使 WHERE 條件永遠為真 → 繞過登入或竊取資料。
// 正常查詢 SELECT * FROM Users WHERE (UserName='fredchen') AND (PassWord='199msq'); // 注入 ' OR '1'='1 後 SELECT * FROM Users WHERE (UserName='' OR '1'='1') AND (PassWord='' OR '1'='1'); // → 條件恆真，繞過驗證

📜 Cross-Site Scripting (XSS)　跨站腳本考過

  • 把腳本注入會被網站「回填到網頁」的輸入；受害者瀏覽器執行該腳本。
  • 經典手法：輸入 <script>alert(document.cookie)</script> → 竊取 cookie。
  • XSS 也可能藏在 e-mail 的超連結中（reflection 反射型）。
🆚 SQL Injection vs XSS
SQL Injection：攻擊「資料庫」（竄改查詢、繞登入、偷資料）；XSS：攻擊「其他使用者的瀏覽器」（執行注入腳本、偷 cookie）。
📄 【2020 B7】Explain (1) SQL Injection and (2) Cross-Site Scripting.2020
▶ 看完整解答
(1) SQL Injection：在輸入欄填入 SQL 片段使查詢語意被竄改，例如輸入 ' OR '1'='1 使 WHERE 永遠為真，繞過登入或竊取資料。
(2) XSS：把腳本（如 <script>alert(document.cookie)</script>）注入會被網站回填到網頁的輸入；受害者瀏覽器執行該腳本（如竊取 cookie）。

電子郵件：SMTP / POP / IMAPE-Mail — 傳遞與文件標準

📧 Delivery vs Document Standards　傳遞標準 vs 文件標準

類型標準用途
傳遞標準 DeliverySMTP伺服器間遞送（寄）郵件，近乎即時
傳遞標準 DeliveryPOP / IMAP收件者稍後下載郵件
文件標準 DocumentRFC 822/2822、HTML、ASCII訊息格式（內容是 ASCII 文字）
  • 每個 client PC 有自己的 mail server；寄信用 SMTP，收信用 POP 或 IMAP。
  • 也可用 Web-based email：全程用 HTTP，訊息文件是 HTML 網頁。
  • ⚠️ 安全：郵件是攻擊的主要載體（病毒、蠕蟲、魚叉式釣魚 spear phishing）。
📄 【卷三 B13(b)】Briefly explain SMTP.考古
▶ 看答案
SMTP：簡單郵件傳輸協定，在郵件伺服器間遞送（寄）電子郵件。

SMTP 指令流程SMTP Command Sequence

📨 The SMTP Dialogue　寄信對話考過

S: EHLO pc-ycc // 打招呼 (舊版 HELO) R: 250 …pleased to meet you S: MAIL FROM: Smith@Alpha.ARPA // (a) 指定寄件者 R: 250 OK S: RCPT TO: Jones@Beta.ARPA // (b) 指定收件者 R: 250 OK S: DATA // 要開始送內文 R: 354 Start mail input; end with <CRLF>.<CRLF> S: Blah blah blah… S: <CRLF>.<CRLF> // (c) 單獨一個句點 = 結束訊息 R: 250 OK S: QUIT // 結束連線
🔑 三個必填空格（考古題 109-1 B8）
(a) = MAIL（組成 MAIL FROM:）；(b) = RCPT（組成 RCPT TO:）；(c) = 單獨一個句點 .（即 <CRLF>.<CRLF> 結束訊息）。
📄 【109-1 B8(1)】SMTP commands (a)(b)(c) for John→Mary.109-1
▶ 看答案
(a)=MAIL（MAIL FROM:）、(b)=RCPT（RCPT TO:）、(c)=單獨一個句點 .（<CRLF>.<CRLF> 結束訊息）。

VoIP：SIP 與 RTPVoice over IP

📞 Signaling vs Transport　信令 vs 傳輸

  • 電信核心概念：Signaling（信令）＝建立連線、拆除、計費等；Transport（傳輸）＝實際語音/影像的傳送。
  • VoIP 省錢：不必維護「語音與資料」兩套網路；語音數位化時還可壓縮。

🔔 SIP — Signaling Protocol　SIP 信令協定🔥 考過2次

  • SIP（Session Initiation Protocol）是 IETF 定義的 VoIP 信令標準。
  • 不只開連線，還負責開/關連線、計費記帳等所有信令工作。
  • 撥號流程：撥號端送 SIP INVITE 給自己的（第一台）proxy → 轉送到對方的（第二台）proxy → 再轉給被叫端；回應沿反向路徑回來。
  • 信令建立連線後，RTP 語音直接在主叫與被叫端之間傳送（不經 proxy）。
信令(藍)經兩台 proxy 建立；建立後語音(綠 RTP)直接在兩端之間傳

🎙️ RTP over UDP　為何用 UDP 不用 TCP

  • 語音是即時的，沒有時間等待重送遺失封包。
  • UDP 輕量、開銷小，適合即時流量；TCP 的重送機制反而拖慢。
  • RTP（Real-time Transport Protocol）再於 UDP 上加 序號（sequence number）與 時間戳（time stamp），補足 UDP 兩個限制。
📄 【109-1 B9】(1) caller→callee 經兩台 proxy 如何撥號？(2) VoIP 用 RTP over UDP，為何不用 TCP？109-1
▶ 看完整解答
(1) 撥號端送 SIP INVITE 給自己的(第一台)proxy，轉送到對方的(第二台)proxy，再轉給被叫端；回應沿反向路徑回來。信令建立連線後，RTP 語音直接在主叫與被叫端之間傳送。
(2) 語音是即時的，沒有時間等待重送遺失封包；UDP 輕量、開銷小，適合即時流量；RTP 再於 UDP 上加序號與時間戳。
📄 【卷三 A14】IETF defines ____ as the signaling protocol of VoIP.考古
▶ 看答案
SIP（Session Initiation Protocol）。

HTTP 與 HTMLHTTP Request / Response

🌐 HTTP Request Message　HTTP 請求訊息🔥 考過2次

GET /panko/home.htm HTTP/1.1[CRLF] Host: voyager.shidler.hawaii.edu Connection: Keep-Alive
  • 第一行：方法（GET）＋ 路徑 ＋ HTTP 版本。
  • Host: 指出要連的主機名（虛擬主機必需）。
  • CR LF＝控制字元（0x0D、0x0A），用來標記文字檔的換行。
📄 【109-1 B8(2)】HTTP request (d)(e) for http://abc.com/news.html.109-1
▶ 看答案
(d)=GET（GET /news.html HTTP/1.1）、(e)=Host（Host: abc.com）。
📄 【卷三 B11】Give an example of an HTTP request message.考古
▶ 看答案
GET /panko/home.htm HTTP/1.1[CRLF] / Host: voyager.shidler.hawaii.edu / Connection: Keep-Alive。

📥 HTTP Response & HTML　回應訊息與下載檔數

HTTP/1.1 200 OK[CRLF] Date: … Server: … Content-type: text/plain[CRLF] [CRLF] // 接著是要下載的位元組（文字/圖/聲音/影片）
  • HTML 檔含文字與「圖片標籤」；瀏覽器依標籤分別下載各圖檔再渲染。
  • 💡 小題：一個網頁有 3 張圖、還會播音樂 → 瀏覽器要下載 1(HTML)+3(圖)+1(音樂)=5 個檔。
  • Keep-Alive：timeout=5（連線開 5 秒）、max=300（最多 300 個連續請求）。

P2P 點對點Peer-to-Peer Applications

🔗 BitTorrent — Tracker & Swarm　tracker 與 swarm考過

  • Tracker（追蹤器）：協助節點找到其他節點的伺服器，記錄哪些 peer 在分享該檔、各擁有哪些片段。
  • Swarm（群）：正在共同分享/下載同一檔案的所有 peer 的集合（種子 + 下載者）。
  • P2P 中，clients 互相提供服務；伺服器只做次要工作（如 tracker 幫忙找對方）。
Tracker 幫各 peer 互相找到（虛線）；Swarm 是分享同檔的所有 peer，彼此交換片段（綠線）
📄 【109-1 B3】BitTorrent: (1) What is a tracker? (2) What is a swarm?109-1
▶ 看答案
(1) Tracker：協助節點找到其他節點的伺服器，記錄哪些 peer 在分享該檔、各擁有哪些片段。
(2) Swarm：正在共同分享／下載同一檔案的所有 peer（種子+下載者）的集合。

📦 Brief Definitions　簡答常見名詞考過

名詞解釋

📄 【卷三 B13】Briefly explain: (a) MTSO (b) SMTP (c) SaaS (d) 1000Base-T.考古
▶ 看答案
(a) MTSO：行動電話交換局，連各 cellsite 並接到 PSTN，負責換手與通道分配。
(b) SMTP：簡單郵件傳輸協定，在郵件伺服器間遞送郵件。
(c) SaaS：軟體即服務，應用以雲端服務提供，經瀏覽器使用。
(d) 1000Base-T：Gigabit Ethernet（1 Gbps），跑在 4 對 UTP、最長 100 m。

互動測驗Self-Check Quiz — 點選作答，立即看詳解

🧪 隨堂測驗（10 題）

1. SMTP 中，結束郵件訊息本文的方式是？
✅ 用 單獨一個句點（<CRLF>.<CRLF>）結束訊息本文；DATA 後開始送內文。
2. 寄信、收信分別用哪些協定？
✅ 寄信 SMTP；收信 POP 或 IMAP。Web mail 則全程用 HTTP。
3. IETF 定義的 VoIP 信令協定是？
✅ SIP（Session Initiation Protocol）負責開/關連線、計費等信令工作。
4. VoIP 的語音傳輸為何用 UDP 而非 TCP？
✅ 即時語音沒時間等重送，UDP 輕量開銷小；RTP 再加序號與時間戳補足。
5. SQL Injection 常見的「' OR '1'='1」會造成？
✅ 使 WHERE 條件永遠為真 → 繞過登入或竊取資料（攻擊資料庫）。XSS 才是偷 cookie。
6. 「<script>alert(document.cookie)</script>」被網站回填到網頁後執行，屬於哪種攻擊？
✅ XSS：注入腳本被回填、由受害者瀏覽器執行（如偷 cookie），攻擊「別人的瀏覽器」。
7. 一個 HTTP 請求訊息的第一行應包含？
✅ 例：GET /panko/home.htm HTTP/1.1，下一行才是 Host:。
8. 網頁有 3 張圖、載入時還播音樂，瀏覽器共要下載幾個檔？
✅ 1(HTML) + 3(圖) + 1(音樂) = 5 個檔。
9. BitTorrent 的 tracker 是什麼？
✅ Tracker 幫 peer 互相找到；Swarm 才是分享同檔的所有 peer 集合。
10. SaaS 指的是？
✅ SaaS（Software as a Service）：應用軟體以雲端服務提供，使用者經網路/瀏覽器使用。

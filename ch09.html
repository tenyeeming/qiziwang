📐 The 2ᴺ − 2 Rule　核心公式🔥 必考計算

📐 公式
某段有 N bits → 可表示 2ᴺ − 2 個子網（或每子網主機數）

「−2」是因為不能全 0（初始未分配）、也不能全 1（廣播）。

  • 公司拿到 Network 部分，把剩下的 bits 切成 Subnet 與 Host。
  • ⚖️ 取捨：Subnet 部分越大 → 子網越多，但 Host 部分越小 → 每子網主機越少（反之亦然）。總和固定。

🧮 Exam Calculation　考古題：49.x 私有網規劃考過

【109-1 B2】公司用私有位址 10.x.x.x：(1) 至少 50 個子網要幾 bits 當 subnet？(2) 每子網幾台主機？

(1) 求 subnet 位元:
　5 bits → 2⁵−2 = 30 < 50 ✗ 不夠
　6 bits → 2⁶−2 = 62 ≥ 50 ✓ → 需 6 bits
(2) 求 host 位元:
　10.x.x.x 是 /8 → 剩 32−8 = 24 bits
　扣 6 bits subnet → host = 18 bits
　主機數 = 2¹⁸−2 = 262,142 台
💡 解題順序
① 先用 2ᴺ−2 ≥ 需求，找最小的 subnet 位元；② host 位元 = (32 − network) − subnet；③ 再套 2ᴺ−2 算主機數。
📄 【109-1 B2】10.x.x.x，(1) ≥50 子網要幾 bits？(2) 每子網幾台主機？109-1
▶ 看答案
(1) 6 bits（2⁶−2=62≥50；5 bits 只有 30 不夠）。
(2) /8 剩 24 bits，扣 6 → host=18 bits → 2¹⁸−2 = 262,142 台。
📄 【2020 A4】The subnet part is 6 bits long. Company A may have at most ____ subnets.2020
▶ 看答案
62 個（2⁶−2=62；扣全 0 與全 1）。
📄 【卷三 A15】If the host part is 6 bits, the maximum number of hosts in the subnet is ____.考古
▶ 看答案
62（2⁶−2）。同樣公式套用在 host 部分。

NAT 網路位址轉換Network Address Translation

🔄 How NAT Works (Translation Table)　NAT 轉換表運作考過

  1. 外送封包：NAT 防火牆把內部的「真實來源 IP + port」換成對外的「假 IP + port」，並把對應關係記入轉換表（translation table）。
  2. 回送封包：依轉換表反查，把對外假 IP/port 還原成原本的內部 IP/port。
內部真實 IP:port ↔ 對外假 IP:port，記在轉換表中雙向對應

🎯 Why NAT?　NAT 的兩大用途

  • ① 安全：外部嗅探者只看得到「對外的假 IP」，查不到內部主機真實位址，無法直接攻擊內部。
  • ② 擴充位址：每個對外 IP 約有 4,000 個 ephemeral port → 一個 IP 可支撐約 4,000 條對外連線；給 248 個 IP 約可有上百萬條連線。
  • 透明性：NAT 防火牆做完所有工作，內外主機都不知道有 NAT，毋需修改主機。
  • ⚠️ 副作用：IPsec、VoIP 等對 NAT 穿越（traversal）較棘手，需特別處理。
📄 【卷三 B12】Describe NAT operation in terms of the translation table.考古
▶ 看答案
外送封包：NAT 把內部來源 IP+port 換成對外(假)IP+port，並把對應關係記入轉換表。回送封包：依轉換表反查還原成原本的內部 IP:port。對內外主機都透明，可擴充位址數並增加安全性。

DNS 網域名稱系統Domain Name System

🌐 What DNS Does　DNS 做什麼考過

  • 把主機名稱（host name）轉成 IP 位址。例：瀏覽器拿到 www.abc.com → 用 DNS 查出 IP。

VPN 與 IPsecVPNs & IP Security

🔐 VPN Basics　VPN 基礎

  • VPN（虛擬私有網路）＝在不受信任的網路（Internet、無線）上做加密保護的通訊；「私有」指別人看不到內容。
  • 三種：Remote access VPN（遠端使用者連到公司站點的 VPN gateway）、Site-to-Site VPN（兩站間所有流量都保護，各站有 gateway 加解密）、Host-to-Host VPN。

🛡️ IPsec　IP 安全標準

  • IETF 的 IP 安全標準，運作在網際網路層（Layer 3）。
  • 保護傳輸層標頭、應用內容、以及至少部分 IP 標頭內容。
  • 保護是「透明的」：上層內容根本不知道自己正被保護。
  • 用 ESP（Encapsulating Security Payload）提供機密性、認證、訊息完整性。
📌 IPsec 兩種模式
Transport mode（傳輸模式）與 Tunnel mode（隧道模式）——下一節是全章最常考的比較。

IPsec：Transport vs TunnelTransport Mode vs Tunnel Mode — 全章最常考

⚖️ The Core Comparison　核心比較🔥 考過3次

項目Transport 傳輸模式Tunnel 隧道模式
保護範圍主機到主機：Internet ＋ 站內網路都保護只保護 Internet 段，站內網路不保護
誰需要憑證/設定每一台主機都要數位憑證與設定只有兩端 gateway 需要憑證與設定
安全性較佳（host-to-host）；但防火牆讀不到加密內容較差（只保護 Internet 段）
成本較高（每台主機都要設定）較低（只在 gateway 運作）
Transport：每台主機都要憑證、全程保護、貴；Tunnel：只有 gateway 要憑證、只護 Internet、便宜
🔑 一句話結論
安全：Transport 勝；成本：Tunnel 省。Tunnel「只有兩端 gateway 需憑證」是最常考的填充。
📄 【109-1 A7】In the ____ mode of IPsec, only the two IPsec gateways need digital certificates.109-1
▶ 看答案
Tunnel（隧道模式；只有兩端 gateway 需憑證與設定）。
📄 【2020 B9】Compare IPsec Transport mode and Tunnel mode in terms of security and cost.2020
▶ 看完整解答
Transport：保護「主機到主機」，涵蓋 Internet 與站內；每台主機都需憑證與設定 → 安全性較佳但成本較高（且防火牆讀不到加密內容）。Tunnel：只有兩端 gateway 需憑證/設定，只保護 Internet 段、站內不保護 → 成本較低但安全性較差。結論：安全 Transport 勝；成本 Tunnel 省。
📄 【卷三 A13】IPsec's ____ mode provides no security in the site network and needs no digital certificate in each host.考古
▶ 看答案
Tunnel（隧道模式）。

  • IPv6 位址共 128 bits；Interface ID 固定 64 bits（和 IPv4 host 部分可變不同）。

ICMP 與 PingInternet Control Message Protocol

📡 ICMP　網際網路控制訊息協定考過

  • 處理不同主機/路由器之間「網際網路層程序」的監督性通訊，避免讓 IP 自己處理這類訊息。
  • 提供錯誤通告（error advisement）訊息，協助診斷錯誤，但不提供可靠性。
  • Ping：告訴發送者對方是否可達，並給出往返延遲（round-trip latency）。Ping 使用 ICMP。
📄 【109-1 A6】____ is a protocol, usually used by routers, to send error or supervisory messages.109-1
▶ 看答案
ICMP（Internet Control Message Protocol）。

互動測驗Self-Check Quiz — 點選作答，立即看詳解

🧪 隨堂測驗（10 題）

1. 子網部分有 6 bits，最多可有幾個子網？
✅ 2⁶ − 2 = 62（扣掉全 0 與全 1）。
2. 公司用 10.x.x.x（/8），要至少 50 個子網，subnet 部分需幾 bits？
✅ 5 bits 只有 2⁵−2=30 不夠；6 bits 有 2⁶−2=62 ≥ 50。
3. 下列哪一個是私有 IP 位址？
✅ 192.168.10.201（192.168/16）。三段私有：10/8、172.16–31、192.168/16；100、72、19 都是公有。
4. NAT 防火牆「回送封包」時做什麼？
✅ 外送時換成假 IP:port 並記表；回送時依表反查還原成內部真實位址。
5. IPsec 哪種模式「只有兩端 gateway 需要數位憑證」？
✅ Tunnel（隧道模式）：只有兩端 gateway 要憑證、只保護 Internet 段、成本較低。
6. 關於 IPsec Transport vs Tunnel，何者正確？
✅ Transport：主機到主機、護 Internet＋站內、每台主機要憑證 → 較安全但較貴。Tunnel 較便宜但較不安全。
7. 瀏覽器拿到 www.abc.com，要查它的 IP 位址，用哪個協定？
✅ DNS：主機名 → IP。（ARP 是 IP → MAC，別搞混）
8. ping 指令使用哪個協定？
✅ ICMP：ping 告知對方是否可達並給往返延遲。ICMP 提供錯誤/監督訊息但不提供可靠性。
9. 下列哪個是「外部（exterior）」動態路由協定？
✅ BGP（Border Gateway Protocol）是外部路由協定；OSPF、EIGRP 是內部。
10. IPv6 全域單播位址的 Interface ID 長度是？
✅ Interface ID 固定 64 bits（與 IPv4 host 部分可變不同）；前綴+子網 m+n=64。
